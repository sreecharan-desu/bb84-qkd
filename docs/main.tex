\begin{abstract}
\noindent This project report details the development and evaluation of an Adversarial Error-Classification Engine (\textit{AECE}) designed for Quantum Key Distribution (QKD) security enhancement. Current QKD implementations rely primarily on a static 11\% Quantum Bit Error Rate (QBER) threshold for security. This project demonstrates a novel approach using interspersed Sentinel Qubits to analyze temporal and spatial correlations of channel errors. By generating high-dimensional "Error Fingerprints," the AECE successfully identifies sub-threshold eavesdropping activity. Computational results show that our framework improves the effective Secure Key Rate (SKR) by recovering high-noise blocks that are statistically uncompromised, proving that traditional scalar constraints are insufficient for maximizing quantum throughput.
\end{abstract}

\section{Introduction}

This project focuses on the practical security of the BB84 protocol, a cornerstone of Quantum Key Distribution (QKD) \cite{bb84}. While BB84 offers unconditional security in theory, physical layer implementations are susceptible to noise and sophisticated eavesdropping strategies that can mimic natural decoherence.

The primary objective of this project is to move beyond the traditional 11\% QBER abort threshold. We introduced a "Sentinel Stream"—a set of non-cryptographic qubits—that serves as an environmental benchmark. By comparing the error statistics of the key qubits against this sentinel baseline, we can detect whether a high QBER is caused by an adversary (Eve) or simply by a noisy physical channel.

\section{System Architecture}

The project simulation models a standard BB84 protocol over a fiber-optic channel exhibiting thermal depolarization and Markovian noise bursts.

\subsection{Threat Model: The Adaptive Adversary}
We simulate an intelligent Eve who performs Intercept-Resend (IR) attacks but limits her activity to remain below the 11\% threshold. This makes her activity indistinguishable from noise when looking at a single QBER scalar, a variation of the Photon Number Splitting (PNS) threat model \cite{pns}.

\section{Mathematical Methodology}

\subsection{Error Delta Extraction}
The system calculates the autocorrelation $R(\tau)$ of the error sequences. The \textit{Error Delta} is defined as the difference between the key error correlation and the sentinel error correlation:
\begin{equation}
\Delta R(\tau) = R_{\text{key}}(\tau) - R_{\text{sentinel}}(\tau)
\end{equation}

\subsection{Risk Assessment via Hoeffding Bounds}
To handle finite-key effects characteristic of practical systems \cite{finite_key}, we apply the Hoeffding bound to establish the probability that an observed delta is a statistical anomaly or a genuine detection of intrusion \cite{hoeffding}:
\begin{equation}
\text{Pr}(|\Delta R_{\text{obs}}| \ge \epsilon) \le 2\exp(-2N\epsilon^2)
\end{equation}

\section{AECE: Machine Learning Engine}

We implemented a Random Forest classifier to process these fingerprints, drawing from recent advancements in ML-based QKD security \cite{ml_qkd}. The machine learning model captures non-linear relationships between autocorrelation, burstiness, and the Error Delta.
